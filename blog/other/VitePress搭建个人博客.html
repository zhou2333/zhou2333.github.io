VitePress搭建部署 ​

官方文档

VitePress 是一个现代化的静态网站生成框架，它是基于 Vite 构建工具的一个插件，旨在简化文档和博客的创建。本篇文章带大家快速搭建一个基于VitePress的静态博客网站并且部署到GitHub Pages上。

前提条件：Node.js 版本大于等于 18

本地VitePress搭建 ​

1. 安装VitePress ​

shell
npm add -D vitepress
npm add -D vitepress
1

2. 构建基本项目 ​

shell
mkdir blog && cd blog
npx vitepress init

部署至GitHub Pages ​

经过上面三个简单的步骤，我们已经可以在本地启动并浏览默认的博客页面了。现在来看如何将我们的博客部署到GitHub上，让任何人都可以通过域名访问你的博客。

1. GitHub新建远程仓库 ​

在GitHub上创建新的仓库，并且仓库名一定是 {username}.github.io。

2. 编写构建及推送脚本 ​

在blog目录下编写脚本

shell
# 编写deploy脚本，脚本代码在下面👇
vim deploy.sh
# 运行脚本
sh deploy.sh
# 编写deploy脚本，脚本代码在下面👇
vim deploy.sh
# 运行脚本
sh deploy.sh

deploy.sh

shell
#!/usr/bin/env sh

# 脚本在执行过程中一旦出现错误，就立即退出
set -e

# 构建
npm run docs:build

# 进入构建结果输入目录
cd docs/.vitepress/dist

# 初始化本地git，初始分支名和远程分支一样，并提交commit
git init -b master
git add -A
git commit -m 'deploy'

# 将本地master分支强制推送到远程master
git push -f git@github.com:zhou2333/zhou2333.github.io.git master

# 返回上一个目录
cd -
